Вам необходимо разработать стратегию защиты данных в рамках всей системы. При этом важно учесть особенности шифрования данных, которые находятся в разных состояниях — при хранении и во время передачи.

1. Определить, какие данные нужно защитить.

2. Реализовать базовые инструменты защиты.

   Предложить средства для контроля за данными

### Профили риска (пояснения к классификации)

**Высокий риск:**

- Нарушение ФЗ-152 "О персональных данных"
- Утечка медицинских данных
- Репутационные потери
- Штрафы регулятора

**Средний риск:**

- Несанкционированный доступ к финансовым данным
- Нарушение коммерческой тайны

**Низкий риск:**

- Нарушение конфиденциальности служебной информации

## 2. Инструменты защиты данных с учетом слоев и методов защиты

### Для данных в покое (Data at Rest) - по слоям данных

**Сырые данные (Raw Data Layer):**

- **Обфускация**: Tokenization для номеров карт пациентов, скремблирование ФИО при передаче в аналитику
- **Шифрование**: AES-256 для Excel файлов с исходными данными, BitLocker FDE для хранилищ
- **Маскировка**: Статическая маскировка для тестовых сред разработки

**Обработанные данные (Processed Data Layer):**

- **Тегирование**: Автоматическое присвоение тегов PII, PHI, Financial для структурированных данных
- **Детерминированное маскирование**: Сохранение связей между данными в разных системах (1С ↔ CRM)
- **Column-level encryption**: Шифрование конкретных полей с медицинскими данными

**Золотой источник (Golden Source):**

- **Transparent Data Encryption (TDE)**: Полное шифрование основной базы данных
- **HSM-управляемые ключи**: Аппаратные модули безопасности для критических ключей
- **Data Lineage контроль**: Отслеживание происхождения каждой записи

### Для данных при передаче (Data in Transit)

**API и интеграции:**

- **mTLS с проверкой сертификатов**: Взаимная аутентификация для API лаборатории
- **OAuth 2.0 + PKCE**: Безопасная авторизация для мобильных приложений
- **Field-level encryption**: Шифрование отдельных полей в JSON при API-вызовах

**Внутрисетевой трафик:**

- **IPSec для tagged данных**: Автоматическое шифрование трафика с тегами PII/PHI
- **TLS 1.3 везде**: Принудительное шифрование всех соединений
- **Data in motion tokenization**: Замена чувствительных данных токенами при передаче

### Для данных в использовании (Data in Use)

**Application-level security:**

- **Гомоморфное шифрование**: Для аналитики без расшифровки (будущее развитие)
- **Trusted Execution Environments**: Изолированная обработка медицинских данных
- **Runtime Data Masking**: Динамическая маскировка в зависимости от роли пользователя

## 3. Средства автоматизированного контроля с тегированием

### Платформы управления данными

**Collibra Data Intelligence Platform:**

- **Автоматическое обнаружение**: Сканирование файловых систем и баз данных на предмет PII/PHI
- **Data Lineage визуализация**: Отслеживание потоков данных от Excel до аналитики
- **Policy automation**: Автоматическое применение политик шифрования на основе тегов

**BigID Deep Data Intelligence:**

- **ML-классификация**: Автоматическое присвоение тегов медицинским данным
- **Risk scoring**: Оценка рисков утечки для каждого набора данных
- **Compliance mapping**: Соответствие тегов требованиям ФЗ-152

**Microsoft Purview (локальная альтернатива):**

- **Data Discovery**: Поиск незащищенных персональных данных
- **Sensitivity labeling**: Автоматическая разметка документов
- **DLP с тегами**: Предотвращение утечек на основе меток данных

### Мониторинг с учетом тегов

**Tag-aware DLP:**

- Блокировка передачи данных с тегами PII/PHI через незащищенные каналы
- Автоматическое шифрование email с медицинскими данными
- Контроль копирования файлов с критическими тегами

**RBAC + ABAC интеграция:**

- Доступ к данным на основе ролей И атрибутов (время, местоположение, устройство)
- Динамические права доступа в зависимости от тегов данных
- Zero Trust для данных с тегом "Critical"

**Audit по тегам:**

- Отслеживание всех операций с данными, имеющими теги PII/PHI
- Алерты при попытке доступа к критическим данным вне рабочего времени
- Автоматические отчеты о соответствии для регуляторов

### Автоматизированные действия на основе тегов:

**Lifecycle Management:**

- Автоматическая архивация данных с тегом "Expired"
- Удаление тестовых данных с тегом "Temp" через 30 дней
- Бэкап критических данных в зашифрованном виде

**Encryption triggers:**

- Автоматическое шифрование при присвоении тега PII
- Усиленная защита для данных с тегом PHI
- Tokenization при передаче данных с тегом "Financial"

**Compliance automation:**

- Автоматическая подготовка отчетов по GDPR/ФЗ-152
- Блокировка обработки данных при отзыве согласия
- Автоматическое обезличивание для аналитики
